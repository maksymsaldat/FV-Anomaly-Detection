## Project Overview

This repository contains the complete implementation developed for the Master's Thesis focused on the application of **Machine Learning techniques for predictive monitoring of photovoltaic power plants**.

The proposed methodology combines data preprocessing, machine learning, statistical analysis and anomaly classification to monitor photovoltaic power generation and detect abnormal operating conditions.

The system is capable of:

- Predicting the expected AC power generation.
- Detecting anomalies based on prediction errors.
- Classifying anomalies according to their severity.
- Grouping consecutive anomalous records into operational events.
- Visualising the results through a monitoring dashboard.

---

# Objectives

The main objective of this project is to develop an intelligent monitoring system capable of assisting photovoltaic plant operators by automatically detecting abnormal production behaviour.

The complete workflow includes:

1. Exploratory Data Analysis (EDA)
2. Data preprocessing
3. Machine Learning model training
4. Expected power prediction
5. Statistical anomaly detection
6. Severity classification
7. Event generation
8. Dashboard visualisation

# Methodology

The proposed methodology is organised into five independent stages.

## Exploratory Data Analysis

* Dataset inspection
* Missing value analysis
* Descriptive statistics
* Correlation analysis
* Data quality verification

---

## Data Preprocessing

The preprocessing stage performs:

* Plant-level aggregation
* Data cleaning
* Meteorological data integration
* Feature engineering
* Night-time filtering
* Export of the final modelling dataset

Selected variables:

* AC_POWER
* IRRADIATION
* AMBIENT_TEMPERATURE
* MODULE_TEMPERATURE
* HOUR

---

## Machine Learning Prediction Model

A **Random Forest Regressor** was trained to estimate the expected power generated by the photovoltaic plant.

### Input variables

* Irradiation
* Ambient temperature
* Module temperature
* Hour

### Output variable

* AC_POWER

The model was trained using Plant 1 and externally validated using Plant 2.

---

## Anomaly Detection

Expected production generated by the Machine Learning model is compared against observed production.

The methodology includes:

* Prediction error calculation
* Relative error calculation
* Moving average smoothing
* Statistical threshold estimation (95th percentile)
* Automatic anomaly detection

---

## Severity Classification

Detected anomalies are classified into four severity levels.

| Severity | Criterion                             |
| -------- | ------------------------------------- |
| Normal   | Error ≤ Threshold                     |
| Low      | Threshold < Error ≤ 2 × Threshold     |
| Medium   | 2 × Threshold < Error ≤ 3 × Threshold |
| High     | Error > 3 × Threshold                 |

Consecutive anomalous records are automatically grouped into operational events.

Each event stores:

* Start time
* End time
* Duration
* Mean error
* Maximum error
* Severity level
